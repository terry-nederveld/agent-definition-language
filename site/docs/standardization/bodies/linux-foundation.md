# Linux Foundation / AAIF

The Linux Foundation (LF) hosts several standards-related projects, including the OpenAPI Initiative (OAI) and the emerging Agentic AI Foundation (AAIF). This document outlines the pathway for ADL standardization through LF projects.

:::tip Why Linux Foundation?
The Linux Foundation's track record with OpenAPI and AsyncAPI makes it a natural fit for ADL. Community-driven governance and industry adoption align with ADL's goals.
:::

## Overview

| Attribute | Details |
|-----------|---------|
| Organization | Linux Foundation |
| Relevant Initiative | AAIF (Agentic AI Foundation) / OAI-style |
| Typical Process | Project proposal, Technical Charter, Working Group formation |
| IPR Model | Apache 2.0 / Community Specification License |
| Timeline | 6-18 months to establish, ongoing development |

## Why Linux Foundation?

- **Community-driven:** Open governance with industry participation
- **Established ecosystem:** Proven track record with OpenAPI, AsyncAPI, GraphQL
- **Industry adoption:** Major technology companies participate
- **Tooling ecosystem:** Strong support for tooling and implementations

## Potential Pathways

### 1. AAIF (Agentic AI Foundation)

If AAIF is established, ADL could be proposed as a specification project:

- Aligns with agent-focused mission
- Potential synergy with other agent protocols
- Industry consortium backing

### 2. OpenAPI Initiative Adjacent

ADL could be proposed as a related specification:

- Similar structure to OpenAPI/AsyncAPI
- Complements existing API specifications
- Leverages existing OAI governance model

### 3. New LF Project

Standalone project with dedicated governance:

- Full control over specification direction
- Requires broader community building
- Longer establishment timeline

## Current Status

- **Phase:** Community Draft
- **LF Contact:** Not yet established
- **Working Group:** Not yet formed
