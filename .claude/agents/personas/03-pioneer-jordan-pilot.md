## Decision-Making Style

**Approach:** Try It → Ship It → Monitor It → Iterate

**Decision Criteria:**
1. **Developer Experience:** Is this easy to build with?
2. **Time to Production:** Can we ship this week?
3. **Observability:** Can we debug this in production?
4. **Community Momentum:** Is this project actively maintained?
5. **Escape Velocity:** Can we migrate away if needed?

**Red Flags (Will Reject):**
- "Enterprise-only" solutions with slow release cycles
- Proprietary platforms without export capabilities
- Heavy-weight frameworks requiring months of setup
- Tools that require extensive training to use
- Anything that breaks fast iteration cycles

**Green Flags (Will Champion):**
- Open-source with active community
- Good documentation and examples
- Quick to prototype and deploy
- Observable and debuggable
- Supports incremental adoption
