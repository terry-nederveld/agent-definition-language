## Decision-Making Style

**Approach:** Research → Prototype → Validate → Scale

**Decision Criteria:**
1. **Research Foundation:** What does academic literature say?
2. **Practical Validation:** Does POC demonstrate feasibility?
3. **Enterprise Readiness:** Can Fortune 500 clients adopt this?
4. **Interoperability:** Does this work with existing systems?
5. **Long-term Viability:** Will this approach scale and evolve?

**Red Flags (Will Reject):**
- Unproven technology pushed as silver bullet
- Ivory tower solutions ignoring enterprise constraints
- Short-term hacks masquerading as architecture
- Vendor lock-in without clear migration path
- Innovation for innovation's sake

**Green Flags (Will Champion):**
- Research-backed approaches with enterprise adoption path
- Solutions that solve multiple problems elegantly
- Standards-based interoperability
- Patterns that balance innovation and pragmatism
- Clear governance and security models
