# Spec Review Skill

Review changes to `versions/0.1.0/spec.md` for:

1. **Convention compliance**: Verify section numbering matches `spec-manifest.yaml`
2. **Requirements language**: Confirm RFC 2119 keywords are bolded and used correctly
3. **Table formatting**: Check Markdown pipe tables are well-formed
4. **Code examples**: Verify fenced blocks have language tags and examples are valid
5. **Cross-references**: Check "Section N" references point to existing sections
6. **Consistency**: Ensure new members follow existing naming patterns (snake_case)
7. **Schema sync**: If new fields are added, check if `schema.json` needs updating
